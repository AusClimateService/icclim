Welcome to ICCLIM’s documentation!¶

Index Calculation CLIMate is a Python libary designed for calculation of climate indices and indicators. ICCLIM calculates currently the ECA&D climate indices. Other climate indices and indicators could be added also.

NEWS:

27/01/2015

  • New version: 3.0
  • icclim.indice() is now used for all indices (simple, multivariable, percentile-based, etc)
  • Added seasonal temporal aggregations: “ONDJFM”, “AMJJAS”, “DJF”, “MAM”, “JJA” and “SON”.

24/10/2014

  • New version: 2.2
  • Added compound percentile-based indices: CD, CW, WD and WW
  • Percentiles are now computed by C function to take into account fill_values

9/10/2014

  • It is possible now to do regridding (works only with rectangular “lat/lon” grid).

8/08/2014

  • icclim.get_percentile_dict() works now with OPeNDAP datasets (added the transfer_limit_bytes parameter)

4/08/2014

  • Improved callback in icclim.indice(), icclim.indice_multivar() and icclim.indice_perc()

31/07/2014

  • Possibility to save a dictionary with daily percentiles in file (added the save_to_file parameter in icclim.get_percentile_dict())

21/07/2014

In icclim.indice(), icclim.indice_multivar() and icclim.indice_perc():

  • Removed the project parameter
  • The time_range parameter is not required, i.e. if it is None, then the whole period of input files will be processed

17/07/2014

  • Added percentile indices
  • Added utility functions for spatial statistics
  • The documentation is updated
